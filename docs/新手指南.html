SkillsBuilder × Tool-Calling 搭配使用 SOP
小白也能照著操作的完整教學

💡 先記住一個比喻就好

專案 角色 做什麼
SkillsBuilder 🗄️ 應用商店 / 食材庫 收集、分類、治理 93 個技能。東西靜靜躺在那裡等你用。
Tool-Calling 🎙️ 智能管家 你說「我要做 OO」,它自動找工具、安全執行、用完收拾、還會自我進化。
0

準備你的電腦(只做一次)

你的電腦需要先裝 3 個免費小程式,就像手機要先有電一樣。

要裝什麼?

小程式它是做什麼的去哪下載
Node.js 讓管家能跑起來 nodejs.org — 按大大的綠色 LTS 按鈕
Git 讓電腦能下載專案 git-scm.com — 一直點 Next
Docker Desktop 讓管家有個「安全盒子」執行工具 docker.com — 裝完要重開機

怎麼確認裝好了?

按鍵盤 Win 鍵,輸入 powershell,點開藍色視窗,依序貼上這三行(每貼一行按一次 Enter):

node --version
git --version
docker --version
✅ 每一行都要跑出一串數字(如 v20.11.0、git version 2.45.0、Docker version 24.0),看到數字 = 成功。
⚠️ 如果某一行沒跑出數字、或說「找不到」,代表那個還沒裝好,回去官網重裝。
1

把兩個專案抓到電腦裡

1-1 打開 PowerShell,進到目標資料夾

貼這一行,按 Enter:
cd d:/Self-developed_Apps

(這意思是「進到這個資料夾」。如果你不想放這裡,換成你喜歡的路徑也行。)

1-2 抓第一個專案:Tool-Calling

ℹ️ 你電腦裡已經有這個專案了(就在 d:/Self-developed_Apps/Tool-Calling)。如果它在上面那個資料夾裡,這步可以跳過。

1-3 抓第二個專案:SkillsBuilder

貼這一行,按 Enter:
git clone https://github.com/Chun-Chieh-Chang/SkillsBuilder.git

看到 done = 成功。

1-4 確認兩個專案都到位了

貼這行:
ls d:/Self-developed_Apps
✅ 你應該看到兩個名字:
Tool-Calling 和 SkillsBuilder
兩個都看到 = 完成。
2

讓管家認識 SkillsBuilder 的所有技能(關鍵一步)

這一步是把兩個專案接起來。只要 3 行指令。

2-1 進到管家(Tool-Calling)的資料夾

貼這一行,按 Enter:
cd d:/Self-developed_Apps/Tool-Calling

2-2 跟管家說「SkillsBuilder 是一個技能來源」

貼這一行,按 Enter(會等幾秒鐘):
node cli.js add https://github.com/Chun-Chieh-Chang/SkillsBuilder
✅ 看到這個 = 成功:
✓ 已新增工具: SkillsBuilder (skillsbuilder)

2-3 讓管家把裡面 93 個技能全部掃進腦袋

貼這一行,按 Enter(會等約 10 秒):
node cli.js index-subtools skillsbuilder
✅ 看到這個 = 成功:
✓ 發現 93 個子工具
已將 subTools 寫入資料庫。

2-4 檢查一下(可選)

貼這行:
node cli.js info skillsbuilder
往下捲,看到 子工具 (93): 後面列出一堆技能名字(tavily、graphify…) = 一切正常 ✅
🔁 之後 SkillsBuilder 那邊 git pull 更新後,回 Tool-Calling 重跑 index-subtools skillsbuilder 即可。
3

開始用!(之後每天都這樣)

用法 A:跟 AI 聊天就能用(最簡單,推薦新手)

打開你的 AI 工具(Claude、Cursor、ChatGPT 桌面版…都行),直接打字:

「啟動全自動工具調用模式,我想要 ______」

例如:

  • 「啟動全自動工具調用模式,我想要做一份投影片」
  • 「啟動全自動工具調用模式,我想要查某個主題的資料」
  • 「啟動全自動工具調用模式,我想要把程式碼畫成關係圖」
AI 會告訴你它找到哪個工具、列出想執行的指令,問你「可以嗎?」你回「可以」→ 它自動執行、用完收拾。
你全程不用打任何指令,只要聊天 + 確認「可以」。

用法 B:自己用鍵盤操作(想自己掌控時用)

在 PowerShell 裡,照順序貼這 5 行(每行按 Enter,等它跑完再貼下一行):

① 進到管家資料夾
cd d:/Self-developed_Apps/Tool-Calling
② 搜尋 — 用白話講你想要什麼
node cli.js search "我想做投影片"

👉 會列出最適合的工具(看信心度那條綠色長條,越長越適合)

③ 看詳情 — 確認這工具適不適合
node cli.js info ppt-master

👉 注意看「🚫 禁用場景」——如果你的需求踩到紅線,換一個工具

④ 安裝 — 把工具抓下來,放在暫存區
node cli.js install ppt-master
⑤ 執行 — 在安全盒子裡跑,不怕中毒
node cli.js invoke ppt-master
⑥ 收拾 — 用完清乾淨
node cli.js cleanup ppt-master
📊

整條流程長這樣

┌─────────────────────────────────────────────┐ │ 準備電腦 → 裝 Node.js / Git / Docker │ ├─────────────────────────────────────────────┤ │ 抓兩個專案 → Tool-Calling + SkillsBuilder │ ├─────────────────────────────────────────────┤ │ 接線(做一次) → node cli.js add ... │ │ → node cli.js index-subtools │ ├─────────────────────────────────────────────┤ │ 日常使用(每天) → 喊咒語 或 手動下指令 │ │ → search → info → install │ │ → invoke → cleanup │ └─────────────────────────────────────────────┘
🔧

卡住了?對照這張表

你遇到的狀況怎麼辦
PowerShell 說「node 找不到」 Node.js 沒裝好。重裝後關掉 PowerShell 重開。
index-subtools 卡很久 正常,它在下載 93 個技能,等 1 分鐘就好。
invoke 跳出紅字提到「docker」 Docker 沒開。去開始選單點開 Docker Desktop,等它變綠燈再試。
搜尋找不到東西 換個說法,例如「投影片」改成「簡報」「presentation」。
想看總共有哪些工具 貼 node cli.js list
想離開、重來 關掉 PowerShell 視窗,重開一個就好,不會壞掉。

一句話總結:

裝好 3 個小程式 → 抓兩個專案 → 跑 3 行接線指令 → 以後跟 AI 說「啟動全自動工具調用模式 + 你想要什麼」就好。

接線(第二步)一輩子只要做一次。之後每天用都是第三步那麼簡單。
