🎬 實際開發情境

SkillsBuilder × Tool-Calling 搭配使用完整案例
以「個人作品集網站」專案為例

📋 任務背景

「幫我做一个响应式个人作品集网站，要有暗色系、毛玻璃效果，能展示我的项目、文章、联系方式。」

你是自由接案者。這個案子需要需求分析 → 規劃 → UI 設計 → 實作 → 安全審查 → 外部工具調用等完整流程。以下用三天說明兩個專案如何協同工作。

1

第一天：用 SkillsBuilder 建立工作纪律

🕙 上午 10:00 — 打開 IDE，啟動開發流程

打開你的 IDE（Claude Code / Cursor / Antigravity 任一），SkillsBuilder 已透過 INSTALL.ps1 部署好技能，可直接觸發。

你對 AI 說：
「我要開始做一個作品集網站，幫我啟動開發流程。」

AI 會自動依序觸發 SkillsBuilder 的技能文件：

  • /grill-requirements 先拷問你：客戶有哪些隱藏需求？目標受眾是誰？有沒有參考網站？
  • /planning-with-files 生成 task_plan.md、findings.md、progress.md，追蹤進度。
  • /writing-plans 把模糊的「暗色系、毛玻璃」翻譯成可執行的規格清單。
  • /premium-design 套用 Morandi 色系 + Glassmorphism UI 規範。
  • /tdd-enforcer 強制先寫測試再寫程式碼，杜絕 Vibe Coding。
✅ 到這裡你完全不用自己找工具——SkillsBuilder 的技能文件就是這些 SOP 的 Source of Truth。

🕚 上午 11:00 — 完成第一個頁面後做 Code Review

你對 AI 說：
「幫我審查這段程式碼。」

AI 觸發 /receiving-code-review，以嚴謹態度分析你的程式碼風格、安全性、回歸風險。

🕞 下午 2:00 — 發現不小心寫了 API Key

你發現了這行危險程式碼：
const apiKey = 'sk-1234567890abcdef';  // ← Hardcoded credential
你對 AI 說：
「掃描一下有沒有洩漏的密鑰。」

AI 觸發 /agent-shield，掃描所有 staged 檔案，輸出：

⚠️ Critical: src/config.js:12 — hardcoded API Key detected
🔧 Suggestion: Replace with process.env.API_KEY
⚠️ 這就是 SkillsBuilder 的價值——它不執行工具，但它確保你做每件事的時候都有正確的 SOP 和防呆機制。
2

第二天：用 Tool-Calling 執行具體任務

SkillsBuilder 幫你怎麼做，Tool-Calling 幫你用什麼工具做。今天開始進入「接單 → 選工具 → 沙盒執行」的流程。

🕙 上午 10:00 — 需要生成投影片給客戶看

你對 AI 說：
「啟動全自動工具調用模式，我要做一份投影片給客戶看。」

AI 自動走 AGENTS.md 內建的 SOP：

① 搜尋最適工具
node cli.js search "我要做投影片給客戶看"
#1 PPT Master (ppt-master)          信心度: 92% [L2-keyword]
   ⭐ 場景: Transform meeting notes into polished slide decks
   🚫 禁用場景: Not for video or interactive web slideshows
   🔗 https://github.com/hugohe3/ppt-master

#2 AIPPT                              信心度: 78% [L2-keyword]
#3 NotebookLM2PPT                     信心度: 65% [L3-semantic]
② 查看詳情，確認禁用場景
node cli.js info ppt-master
ID:            ppt-master
URL:           https://github.com/hugohe3/ppt-master
描述:          AI tool converting docs or topics into native PowerPoint decks
分類:          文件生產力
語言:          python
⭐ 場景:       Transform meeting notes into polished slide decks
🚫 禁用場景:   Do not use for creating video content or interactive web-based slideshows
★ 優勢:        Generates fully editable .pptx files
               Supports custom templates, transition animations
               Directly converts text documents into structured outlines
安裝:          pip install ppt-master [pip]
③ 安裝（只抓到 .temp/，不污染系統）
node cli.js install ppt-master
④ Docker 沙盒中執行（完全隔離，不怕中毒）
node cli.js invoke ppt-master
⑤ 用完清理
node cli.js cleanup ppt-master
✅ 整個過程中，ppt-master 只在 Docker 容器裡運行，不會在你的電腦上留下任何相依套件。

🕐 下午 1:00 — 需要分析競品網站架構

你對 AI 說：
「啟動全自動工具調用模式，幫我分析競品的網站架構。」 
node cli.js search "分析競品網站架構"
→ 命中 Playwright（自動化瀏覽器測試工具）

node cli.js install playwright
node cli.js invoke playwright --url https://competitor.com --screenshot
→ 輸出：截圖 + 頁面結構分析報告

🕒 下午 3:00 — 需要畫專案架構圖

你對 AI 說：
「啟動全自動工具調用模式，幫我畫系統架構圖。」 
node cli.js search "畫系統架構圖"
→ 命中 Graphify（知識圖譜工具）

node cli.js install graphify
node cli.js invoke graphify --input ./src --format mermaid
→ 輸出：project-graph.mmd（可在 Mermaid Live Editor 渲染）
3

第三天：兩個專案真正接軌

你用自己的 /skill-creator 寫了一個新技能 my-portfolio-generator，放在 SkillsBuilder 目錄下：

d:/Self-developed_Apps/SkillsBuilder/skills/dev/my-portfolio-generator/SKILL.md

現在你想讓 Tool-Calling 也能搜尋到這個自訂技能：

① 進到 Tool-Calling
cd d:/Self-developed_Apps/Tool-Calling
② 註冊 SkillsBuilder 為工具來源
node cli.js add https://github.com/Chun-Chieh-Chang/SkillsBuilder
③ 深度索引，掃出全部技能
node cli.js index-subtools skillsbuilder
開始深層掃描 Monorepo: SkillsBuilder
正在下載專案...
掃描目錄結構...
✓ 發現 93 個子工具
已將 subTools 寫入資料庫。
④ 驗證
node cli.js search "作品集生成器"
#1 my-portfolio-generator (信心度 85%) [L2-keyword]
#2 ppt-master (信心度 72%)
✅ 你的自訂技能現在也是 Tool-Calling 搜尋庫的一等公民了。SkillsBuilder 負責治理與部署，Tool-Calling 負責檢索與執行。

三天開發流程總覽

Day 1 — SkillsBuilder 管「怎麼做」

/grill-requirements 需求反轉拷問
/planning-with-files 產生任務計劃
/premium-design 套用 UI 規範
/tdd-enforcer 強制 TDD
/agent-shield 安全掃描
/receiving-code-review 程式碼審查

Day 2 — Tool-Calling 管「用什麼工具做」

search 語義搜尋最適工具
info 查看詳細資訊與禁用場景
install 動態安裝到 .temp/
invoke Docker 沙盒隔離執行
cleanup 用完自動清理

Day 3 — 兩者接軌

SkillsBuilder 新增自訂技能 → index-subtools 索引進 Tool-Calling → 自訂技能也能被自動搜尋調用

分工總結

📝 SkillsBuilder

管流程、規範、品質。
確保你在對的時間做對的事。
它是你的 SOP 圖書館。

⚡ Tool-Calling

管檢索、安裝、執行、清理。
讓外部工具自動找到、安全跑完、用完消失。
它是你的工具調度引擎。

開發階段使用專案具體動作
需求分析SkillsBuilder/grill-requirements 拷問模糊需求
專案規劃SkillsBuilder/planning-with-files 產生 task_plan.md
UI 設計SkillsBuilder/premium-design 套用設計規範
程式碼實作SkillsBuilder/tdd-enforcer 強制先寫測試
安全審查SkillsBuilder/agent-shield 掃描硬編碼密鑰
程式碼審查SkillsBuilder/receiving-code-review 嚴謹審計
執行外部工具Tool-Callingsearch → invoke 自動選工具並沙盒執行
畫架構圖Tool-Callinginvoke graphify 產出 Mermaid 圖
生成投影片Tool-Callinginvoke ppt-master 自動安裝、執行、清理

一句話總結：

SkillsBuilder 讓你「知道怎麼做」，Tool-Calling 讓你「自動做到」。

前者是 SOP 圖書館，後者是工具調度引擎。合起來才是完整的 AI 輔助開發閉環。
