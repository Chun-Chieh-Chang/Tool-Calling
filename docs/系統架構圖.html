🔄 技能完整生命週期

1
生產（SkillsBuilder） 開發者使用 /skill-creator 或 /skill-architect 創建 SKILL.md，遵循 YAML frontmatter + MECE 分類規範。
2
審計（SkillsBuilder） /vetter 檢查安全性、/agent-shield 掃描硬編碼密鑰、/complexity-reduction 確保 YAGNI / DRY。
3
部署（SkillsBuilder） INSTALL.ps1 透過 Symbolic Link / Deep Copy 將技能部署到 Claude Code、Cursor、Gemini 等 IDE。
4
索引（Tool-Calling） node cli.js add <url> + index-subtools skillsbuilder 將 SkillsBuilder 的 93 個技能解析進 tools.json 的 subTools 陣列。
5
檢索（Tool-Calling） 用戶發出任務 → L1 精確匹配 → L2 關鍵字 → L3 TF-IDF 語義 → Telemetry 動態加權 → 返回 Top-K 結果。
6
安裝（Tool-Calling） install 將工具原始碼下載至 .temp/<tool-id>（Sparse Checkout 支援），不污染主機環境。
7
執行（Tool-Calling） invoke 在 Docker 沙盒中隔離執行：--network none + --read-only + 512MB 記憶體限制，完全免疫 RCE 風險。
8
清理（Tool-Calling） cleanup 移除 .temp/<tool-id> 臨時目錄，系統復歸乾淨狀態。
9
學習（Tool-Calling） telemetry 記錄執行結果 → 成功率高 → 下次搜尋加分；成功率低 → 扣分。成功軌跡可匯出為 SFT 資料集用於 LLM 微調。

⚡ 日常使用流程（兩種觸發方式）

方式 A：IDE 對話觸發

A1
用戶輸入咒語 「啟動全自動工具調用模式，我要做投影片」
A2
AI Agent 解析意圖 提取關鍵字「投影片」+ 場景「做」
A3
執行 search node cli.js search "做投影片" → 返回 Top 5 候選工具
A4
列出指令等待確認 展示 install / invoke 命令，等用戶說「可以」
A5
自動執行 + 清理 install → invoke（Docker 沙盒）→ cleanup

方式 B：終端機手動操作

B1
搜尋工具 node cli.js search "我要做投影片"
B2
查看詳情 node cli.js info ppt-master（檢查 🚫 禁用場景）
B3
安裝到暫存區 node cli.js install ppt-master
B4
沙盒執行 node cli.js invoke ppt-master
B5
清理臨時檔案 node cli.js cleanup ppt-master

⚖️ 兩專案功能對照表

維度 SkillsBuilder Tool-Calling
定位 靜態技能資產倉庫 + 治理規範 動態工具調度引擎 + 安全執行
核心資料 93 個 SKILL.md（純 Markdown 指引文件） 207 個工具註冊條目（結構化 JSON）
觸發方式 IDE Slash Command：/tavily, /graphify… CLI 指令：node cli.js search/invoke
檢索能力 ❌ 無內建搜尋引擎 ✅ L1/L2/L3 三層漸進式檢索 + Telemetry 加權
執行能力 ❌ 技能為靜態 SOP，不直接執行 ✅ Docker 沙盒隔離執行 + 自動清理
安全機制 /agent-shield 靜態掃描 Docker 隔離 + SAFE_REPO_URL 白名單 + 防注入
學習進化 ❌ 無自我學習機制 ✅ Telemetry 動態加權 + SFT 資料集匯出
IDE 適配 ✅ 十餘種 IDE 適配（Claude/Cursor/Gemini…） ❌ 僅 CLI + Web UI
銜接點 node cli.js index-subtools skillsbuilder → 將 SkillsBuilder 技能索引進 Tool-Calling 搜尋庫

一句話總結：

SkillsBuilder 管「有什麼技能、怎麼治理」，Tool-Calling 管「怎麼找到、怎麼安全執行」。

兩者透過 index-subtools 銜接，構成「技能生產 → 治理 → 部署 → 檢索 → 執行 → 學習」的完整閉環。
